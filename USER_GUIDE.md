# 月之暗面用户指南

这份指南面向普通使用者，说明《月之暗面》v0.3.0-local-stable 的主要界面和功能。应用只做本地记录整理，不提供诊断、治疗建议、医疗建议或危机干预。

## 封面 / 伪装首页

打开应用时默认进入账本式封面。这里可以记录收入、支出和最近账本记录。

从封面右上角设置按钮进入设置，可以通过隐藏入口进入记录界面。记录界面包含成员、群组、私聊、聊天、前台、接续、交接、投票、照护、系统资料和导入导出等功能。

账本记录独立保存在本机，完整 JSON 备份会包含 `ledgerRecords`。

## PWA 安装

如果通过本机 `localhost` / `127.0.0.1` HTTP，或可信的 HTTPS 本地 / 私有静态服务打开，浏览器可能会提供“安装应用”或“添加到主屏幕”。这是可选功能，只是让应用像本机应用一样启动。

PWA 的 Service Worker 只缓存应用静态文件，不缓存导出的 JSON / `.moonenc.json` 备份，不缓存图片 Blob，也不会上传用户数据。聊天、成员、账本、设置等数据仍保存在当前浏览器的 `localStorage` 和 IndexedDB 中。

PWA 不等于云同步，也不替代完整 JSON 备份或加密备份。更新 app 文件后，如果界面还停留在旧版本，可以刷新页面，或清理此站点缓存 / Service Worker 后再打开。直接用 `file://` 打开时，Service Worker 不可用；需要 PWA 离线壳缓存时，请使用本地 HTTP。

## 成员、群组、私聊

侧边栏可以切换：

- 群组：公开或日常对话空间。
- 私聊：私聊 / 小群聊，只控制应用内显示范围。
- 成员：系统成员列表。

可以添加、编辑或删除成员和群组。删除成员不会删除历史消息里的名字快照；删除群组会删除该群组内聊天记录，操作前建议先导出备份。

私聊和小群聊不是加密空间，只是本应用内的可见性和组织方式。

## 聊天与图片

在聊天区选择发言成员和消息分类，然后输入内容发送。

图片发送流程：

- 选择图片后会先进入预览 / 编辑。
- 确认发送后，图片保存到 IndexedDB。
- 消息里长期只保存 `imageId`、图片名和类型。
- 完整 JSON 导出时会把图片补回 DataURL。

消息有校验码序号。清空消息后，序号会从剩余消息重新计算；如果全部清空，下一条消息从 `0001` 开始。

## 前台日志

前台日志用于记录谁在前、谁在场，或者当前状态是否混合 / 模糊 / 未知。

可记录：

- 状态类型
- 在场成员
- 焦点成员
- 记忆清晰度
- 开始时间和结束时间
- 备注

结束时间留空的记录表示仍在进行。应用会尽量保持最多一条进行中的前台记录。

这些记录是自我整理，不是诊断判断。

## 接续面板

接续面板用于帮助后来接手的人快速看到最近情况。

它会汇总：

- 当前或最近前台记录
- 最近交接
- 最近消息
- 议题 / 投票
- 未完成任务
- 照护记录和提醒

可以在设置中启用“进入记录界面时显示接续面板”。接续面板有管理模式，删除来源记录会影响原记录，请谨慎操作。

## 交接与任务

交接用于给后来接手的人留下便签。可以从模板开始，例如睡前、出门、工作 / 学习、情绪强烈后、前台切换等。

保存交接时，可以同时创建接力任务。任务支持：

- 待接力
- 进行中
- 暂停
- 已完成
- 删除

任务可以设置详情、截止时间和适合处理 / 认领成员。

## 投票 / 议题 / 决议

议题 / 投票用于记录内部讨论和决策过程。

可以填写：

- 议题标题
- 议题说明
- 决策方式
- 选项
- 截止时间
- 复盘时间

成员可以投票并填写理由。议题可以暂停、恢复、取消或手动结束。结束后会生成决议文本，并写入聊天和交接。

## 成员档案

成员档案 2.0 支持更多资料：

- 称呼偏好 / 代词
- 别名
- 安抚方式
- 边界
- 需要避免
- 沟通方式
- 前台偏好
- 自定义字段
- 状态历史

成员头像会保存到 IndexedDB，成员数据里长期只保存 `avatarId`。

自定义字段可以设置可见性。visibility 只影响应用内展示和导出，不是加密隔离。

## 高级搜索

高级搜索可以搜索：

- 消息
- 成员资料和部分扩展字段
- 群组 / 私聊
- 交接
- 投票 / 议题
- 前台
- 任务

搜索支持关键词、成员、对话、消息分类、日期范围、是否含图片、当前对话等条件。消息结果可以跳转并高亮。

高级搜索是内存搜索，不是持久化全文索引。数据量很大时可能变慢。

## 隐私桶 / 可见性

系统档案和部分成员字段支持可见性设置，用于控制应用内展示和导出范围。

请记住：

- visibility / 隐私桶不是加密隔离。
- 同一设备、同一浏览器资料里的数据仍在本地。
- 完整 JSON 备份会包含完整数据。
- 脱敏导出只影响当次导出的文本或报告。

## 照护板

照护板 / 需求看板用于记录生活和身体需求，例如饮水、进食、睡眠、疼痛 / 不适、精力、感官负荷、情绪和药物备注。

可以选择把照护记录同时写入当前聊天，写入的消息分类为“状态”。

照护板只是本地生活记录和提醒，不是医疗建议、治疗建议或危机干预。

## 备份健康检查

备份健康检查用于检查图片状态：

- 当前数据引用了哪些图片。
- IndexedDB 中是否缺少这些图片。
- IndexedDB 中是否有不再被数据引用的孤儿图片。

使用顺序建议：

1. 打开设置。
2. 进入“备份健康检查”。
3. 先运行检查。
4. 如果有缺失图片，选择一份完整 JSON 备份。
5. 先预览可修复图片。
6. 确认后再修复。

清理和修复都需要确认，不会自动运行。

## 术语系统

设置中的术语面板可以修改界面常用词，例如成员、系统、前台、共前台、群组、私聊、交接、投票、议题、任务、照护、接续等。

术语只影响界面文案，不修改历史聊天内容，也不改变 JSON 字段名。

## 时间线和月度回顾

时间线会从现有记录中只读聚合：

- 消息
- 前台日志
- 交接
- 议题 / 投票
- 任务
- 照护记录
- 照护清单

可以按日期、成员、对话、当前对话和类型筛选。时间线不聚合账本，不读取图片 Blob，不修改 data。

月度回顾显示本月消息数、前台记录数、交接数、议题 / 投票数、任务数、照护记录数、unknown / blended 前台次数、常见消息分类和成员消息记录次数。

时间线 / 月度回顾只是本地记录统计，不代表状态判断或诊断。

## 复盘报告导出

导出格式中可以选择：

- 复盘报告 Markdown
- 复盘报告 TXT

报告可以按日期范围生成，也可以选择包含哪些章节：

- 概览统计
- 时间线摘要
- 聊天摘要
- 前台记录
- 交接
- 议题 / 投票
- 任务
- 照护记录

复盘报告可以使用脱敏、隐藏成员名和排除私聊选项。脱敏不是 NLP 脱敏，用户导出前仍需人工检查敏感信息，特别是聊天原文、照护 note、投票理由和交接内容。

复盘报告不会导出图片内容，只显示“含图片”。

## 账本伪装首页

封面是一个本地账本界面，可以记录收入、支出、分类、日期和备注。

账本 UI 与记录功能相互独立：

- 账本数据保存在 `moonLedger.records.v1`。
- 完整 JSON 备份包含账本记录。
- 时间线和复盘报告当前不纳入账本记录。

## 导入、导出和分享提醒

完整 JSON 备份包含大量敏感内容和图片 DataURL。请只保存在可信位置，不要随意发送给不可信对象。

普通 Markdown / TXT / CSV 和复盘报告可以使用脱敏选项，但脱敏不是自动隐私审查。分享前请人工检查。
