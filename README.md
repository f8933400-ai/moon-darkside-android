# 月之暗面

《月之暗面》是一个本地离线单页应用，用于多意识体场景下的日常记录、内部沟通、前台日志、交接、任务、议题、照护、时间线回顾和复盘整理。它也带有一个本地账本首页，可作为普通离线账本独立使用。

当前稳定版：`v0.4.0-local-stable`。

本应用只做记录和整理，不提供诊断、治疗建议、医疗建议或危机干预。

## Local-only / Offline-first

- 没有云同步、账号、遥测或远程 API。
- 不需要联网服务，不依赖 CDN、npm 或构建步骤。
- 所有数据默认保存在当前浏览器 / 当前设备内。
- 换浏览器、清理站点数据、重装系统或更换设备前，请先导出完整 JSON 备份。

## 如何打开

最简单的方式是直接打开项目里的 `index.html`。

更推荐用本地 HTTP 服务打开，浏览器兼容性更稳定：

```bash
python3 -m http.server 8080 --bind 127.0.0.1
```

然后在同一台机器的浏览器里打开：

```text
http://127.0.0.1:8080/index.html
```

这只是本机服务，不会把数据上传到外部。

## PWA 本地安装（可选）

在本机 `localhost` / `127.0.0.1` HTTP，或可信的 HTTPS 本地 / 私有静态服务下打开时，浏览器可以把《月之暗面》作为 PWA 安装到桌面或主屏幕。PWA 安装只是更方便地打开应用，不会启用云同步、账号、遥测或远程 API。

Service Worker 只缓存静态应用文件，例如 `index.html`、`styles.css`、`manifest.webmanifest`、本地图标和 `js/` 脚本。v0.4.0 local stable 的静态壳缓存版本是 `moon-app-shell-v0.4.0`，并包含 128 / 192 / 512 本地图标。用户数据仍保存在当前浏览器的 `localStorage` 和 IndexedDB 中；导出的 `.json` / `.moonenc.json` 备份、账本 JSON / CSV、图片 Blob、IndexedDB 内容和本机存储内容不会被 Service Worker 打包或上传。

PWA 不替代完整 JSON 备份或加密备份。更新 app 文件后，Service Worker 会尝试安装新的静态壳缓存并清理旧的 `moon-app-shell-*` 缓存；如果浏览器仍显示旧界面，可以刷新页面，或在浏览器里清理此站点的缓存 / Service Worker 后再打开。

直接用 `file://` 打开时，浏览器不会注册 Service Worker，因此 PWA 离线安装缓存不可用。需要 PWA 能力时，请使用本机 HTTP，或部署到自己信任的 HTTPS 本地 / 私有静态服务。

## 数据保存位置

主数据保存在 `localStorage`，图片保存在 IndexedDB。

- 主数据键：`osddDidLocalJournal.v2`
- 偏好键：`osddDidLocalJournal.prefs.v1`
- 账本键：`moonLedger.records.v1`
- 账本设置键：`moonLedger.settings.v1`
- 图片库：IndexedDB 数据库 `moon-images`，objectStore `images`

新写入的聊天图片、成员头像和房间背景会存进 IndexedDB，主数据里长期只保存图片 ID。完整 JSON 导出时会把图片从 IndexedDB 补回为 DataURL，以便得到单文件备份。

## 功能总览

- 本地账本首页：可记录收入、支出、分类、日期、账户 / 钱包、支付方式和备注，支持分类管理、月度预算、分类预算、CSS 条形图、筛选摘要、操作提示和账本专用 JSON / CSV 导出。
- 成员、群组、私聊：管理成员资料、公开群组和私聊 / 小群聊。
- 聊天记录：按成员身份、分类和房间记录对话，可发送图片。
- 前台日志 2.0：记录前台、共前台、靠近前台、混合 / 模糊、未知 / 不确定等状态。
- 接续面板：进入后汇总最近前台、交接、消息、议题、任务和照护信息。
- 交接与任务：使用交接模板，也可以从交接创建接力任务。
- 投票 / 议题 / 决议：记录讨论、投票理由、暂停 / 恢复 / 取消、决议和复盘时间。
- 成员档案 2.0：扩展资料、自定义字段、状态历史、关系和结构视图。
- 高级搜索：在当前内存数据中搜索消息、成员、房间、交接、投票、前台和任务；它不是持久化全文索引。
- 隐私桶 / 可见性：控制系统资料和部分字段在应用内展示与导出时的可见范围。
- 照护板 / 需求看板：记录身体和生活需求，也可选写入聊天。
- 备份健康检查：检查图片引用、缺失图片、孤儿图片，并支持从 JSON 备份修复缺失图片。
- 可选加密备份：用本机浏览器 Web Crypto 加密完整 JSON 备份，导出为 `.moonenc.json`。
- 自定义术语：把成员、系统、前台、交接、任务、照护、接续等界面词改成自己的说法。
- 时间线总览 + 月度回顾：把现有记录只读聚合成时间线和月度统计。
- 复盘报告导出：按日期范围生成 Markdown / TXT 报告，可选择章节和脱敏选项。

## 备份和恢复

建议定期导出完整 JSON 备份。

1. 打开设置。
2. 进入“导出 / 导入备份”。
3. 导出范围选择“全部群组”。
4. 文件格式选择“JSON 备份”。
5. 保存导出的 `.json` 文件到可信位置。

也可以选择“加密完整 JSON 备份”，导出 `.moonenc.json` 文件。加密完整 JSON 备份固定导出全部主记录数据，不受当前群组 / 指定群组导出范围影响。加密备份使用浏览器 Web Crypto API 在本机完成，不联网；密码不会保存到 `localStorage`，也不会写进备份文件。请务必妥善保存密码，忘记密码后无法恢复加密备份。

加密只保护导出的备份文件，不会加密当前浏览器里的本机数据。普通 JSON 备份仍可使用，也建议至少保留一份安全保存的备份。无论是否加密，备份文件都可能包含敏感记录和图片内容，分享前请谨慎确认对象和场景。

完整 JSON 备份会包含：

- rooms
- members
- messages
- tags
- messageKinds
- polls
- handoffNotes
- frontingLogs
- tasks
- systemProfile
- systemProfileVisibility
- memberRelations
- externalSystemCards
- careLogs
- careChecklist

主记录完整 JSON / encrypted-json 默认不再包含 `ledgerRecords` 或 `ledgerSettings`。账本记录独立保存在 `moonLedger.records.v1`，账本分类和预算设置独立保存在 `moonLedger.settings.v1`，请在首页使用“导出账本备份”或“导出 CSV”单独保存。账本 JSON 备份格式为 `app: "moon-ledger"`、`kind: "ledger-backup"`、`version: 2`、`createdAt`、`records` 和 `settings`，适合完整恢复账本；CSV 仍只导出记录，适合用表格查看。

普通 JSON 的“当前群组 / 指定群组”范围主要用于局部恢复或排查，不是严格脱敏分享文件，也不是完整备份；它可能仍包含全局成员、系统资料或配置。需要完整恢复时请选择“全部群组”或“加密完整 JSON 备份”。

导入 JSON 时，应用会先解析和迁移数据，再把 JSON 中的图片 DataURL 以新的唯一 import 图片 ID 外置回 IndexedDB，避免覆盖本机已有同名图片。导入失败不会覆盖当前 data，并会尽量清理本次导入已经写入的图片。导入旧式 inline 图片备份成功后，会让图片迁移标记回到可重新检查状态；导入含过期投票的主记录备份时，会立即结算到期投票，不再等待定时器。如果原备份中存在 `messageIntegrity` 异常，导入会提示异常数量，异常消息不会被静默洗成“校验正常”。旧版主记录备份如果包含 `ledgerRecords`，新版主导入只会提示，不会自动恢复或覆盖当前账本；如需迁移旧账本，请到账本页使用账本导入功能单独处理。

账本导入不影响记录界面数据、偏好或 IndexedDB 图片库。version 1 账本备份仍可导入，只替换账本记录并保留当前账本设置；version 2 会先校验 settings，再替换账本记录和账本设置。分类改名不会批量迁移旧记录里的 `category` 字符串，预算统计以 `categoryName` 兼容旧记录。账本专用备份也可能包含现实财务信息，请谨慎保存和分享。

导入加密备份时，应用会先在本机用密码解密，成功后再走同一套 JSON 导入流程；图片仍会外置回 IndexedDB。密码错误或文件损坏时不会覆盖当前 data，也不会写入图片库。

P5-03 已验收完整 JSON 和加密完整 JSON 的图片恢复链路：成员头像、房间背景和聊天图片都可以通过完整备份恢复。普通 JSON 的当前群组 / 指定群组范围只恢复该范围内的房间和消息图片；全局成员资料仍按当前导出结构保留。

更多细节见 [BACKUP_GUIDE.md](BACKUP_GUIDE.md)。

## 图片存储说明

- 新聊天图片写入 IndexedDB，只在消息里保存 `imageId`。
- 成员头像写入 IndexedDB，只在成员里保存 `avatarId`。
- 房间背景写入 IndexedDB，只在房间里保存 `backgroundId`。
- 聊天图片、成员头像或房间背景写入后，如果主数据保存失败，应用会尽量删除本次新写入图片并恢复内存状态。
- 删除或替换成员头像、房间背景时，旧图片会延后到主数据保存成功后再清理。
- 完整 JSON 导出会读取 IndexedDB，把图片补回为 DataURL。
- 完整 JSON / 加密完整 JSON 导入后会把这些 DataURL 再外置回 IndexedDB。
- 备份健康检查可以发现缺失引用、孤儿图片，并可从完整 JSON 备份修复缺失图片。
- 时间线和复盘报告不会读取图片 Blob，只显示“含图片”。
- 不建议长期把图片 DataURL 手动塞回主 JSON。

## 隐私说明

本应用不上传数据，但本地隐私仍取决于设备、浏览器资料、备份文件和现实分享方式。

重要限制：

- visibility / 隐私桶只影响应用内展示和导出，不是加密隔离。
- 脱敏导出和复盘报告不会做 NLP 脱敏。导出前请人工检查原文、备注、照护 note、投票理由等敏感信息。
- 复盘报告用于自我整理或与可信对象沟通，不适合直接发给不可信对象。
- JSON 完整备份包含非常完整的数据和图片内容，不要随意分享。
- 加密备份只保护导出的 `.moonenc.json` 文件，不加密当前浏览器存储；设备和浏览器本身仍需要现实层面的保护。
- 进入密码锁只是防止随手查看的本地轻量隐私门帘，不是数据加密；生物识别需要先设置进入密码。需要保护备份文件时，请使用加密完整 JSON 备份。

## 校验码和清空数据

每条消息有一个 `seq`，界面显示为校验码序号，例如 `0001`。清空消息或删除数据后，下一条消息的序号会从剩余消息重新计算；如果消息全部清空，下一条消息会从 `0001` 开始。

P6-01 起，新生成的 `messageIntegrity` 使用真实 U+001F 字段分隔符；旧版本使用字面 `"\\u001f"` 分隔符生成的消息仍会兼容验证，避免旧备份误报。清空或删除数据不会改变校验输入字段集合。

## 已知限制

- 高级搜索是内存搜索，不是持久化全文索引。
- 数据量很大时，localStorage 主数据的加载、保存和搜索可能变慢。
- 聊天和时间线目前没有虚拟滚动。
- visibility 不是加密隔离。
- 加密备份忘记密码无法恢复。
- 复盘报告脱敏不是 NLP 脱敏。
- 照护板不是医疗建议、治疗建议或危机干预。
- 时间线 / 月度回顾只是本地记录统计，不代表状态判断或诊断。
- 没有云同步、账号、遥测或远程 API。

## 故障排查

如果页面打不开：

- 优先用本地 HTTP 服务打开，而不是直接双击文件。
- 确认所有 `js/` 文件、`styles.css`、`app_icon.png`、`app_icon_192.png`、`app_icon_512.png`、`favicon.ico` 都和 `index.html` 在同一项目目录内。

如果图片显示缺失：

- 打开设置里的“备份健康检查”。
- 先运行检查。
- 如果有完整 JSON 备份，可以选择 JSON，预览可修复图片，再确认修复。

如果导入失败：

- 当前 data 不会被覆盖。
- 确认选择的是《月之暗面》导出的 JSON 备份。
- 如果 JSON 文件来自不可信来源，不建议导入。

如果浏览器数据被清空：

- 只能从之前导出的完整 JSON 备份恢复。
- 浏览器无痕模式、清理站点数据、换浏览器都可能让本机数据不可见。
